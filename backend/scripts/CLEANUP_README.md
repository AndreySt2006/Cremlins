Этот README описывает какие файлы в `backend/` считаются безопасными для удаления и какие — опциональные/рискованные.

Безопасно удалять (скрипт `cleanup_backend.sh` удалит эти автоматически):
- `__pycache__/` директории и `.pyc` файлы — артефакты интерпретатора.
- `dev.db` — локальный sqlite файл для разработки (если вы не используете sqlite как единственное хранилище).
- файлы в `static/uploads/` — локально загружённые изображения (если вам не нужны сохранённые изображения).

Опционально удалить (сначала сделайте бэкап, особенно `kremls_db.dump`):
- `../kremls_db.dump` — дамп базы данных (может быть большой). Удалять только если у вас есть другой бэкап или вы уверены, что он не нужен.
- `backend/backend/alembic/versions/` — миграции alembic. Удаление сделает невозможным применить/откатить миграции, но если вы не используете alembic — можно удалить.
- `backend/backend/alembic/README`, `script.py.mako` — небольшие вспомогательные файлы; их можно удалить, но обычно они не мешают.

Рекомендации:
1) Запустите скрипт `cleanup_backend.sh` (из каталога `backend`) чтобы очистить только безопасные артефакты:
   cd backend
   chmod +x scripts/cleanup_backend.sh
   ./scripts/cleanup_backend.sh

2) Если хотите удалить дамп или миграции — сначала сделайте резервную копию (переместите в `archive/`), например:
   mkdir -p ../archive
   mv ../kremls_db.dump ../archive/
   mv backend/alembic/versions ../archive/alembic-versions-backup

3) После удаления проверьте `git status` и убедитесь, что в коммит попали только ожидаемые изменения.

Если хотите — могу автоматически подготовить и выполнить вариант, который удалит и дамп и папку миграций (или только дамп). Скажите точно какие файлы вы хотите удалить, и я подготовлю или выполню команды.
